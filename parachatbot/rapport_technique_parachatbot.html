1. C'est quoi ce projet ?

Ce projet est un module PrestaShop qui ajoute un chatbot intelligent sur le site e-commerce de la parapharmacie Kingphar.ma (Marrakech).

Le chatbot s'affiche sur toutes les pages du site. Le client peut écrire en français, en arabe ou en darija (dialecte marocain). Le chatbot comprend la demande, cherche les produits dans la base de données, et utilise Google Gemini (Intelligence Artificielle) pour formuler une réponse adaptée avec des recommandations de vrais produits.

Exemple : Un client écrit "wejhi fih lhboub" (darija = "j'ai des boutons au visage").
→ Le chatbot traduit → cherche les produits anti-acné dans la base de données → demande à l'IA Google Gemini de formuler une réponse → affiche la réponse + les produits avec images et prix.

Importation des produits : Le vrai site Kingphar.ma contient environ 2600 produits. J'ai développé un script PHP intelligent qui extrait automatiquement tous ces produits (noms, descriptions, prix, images) depuis le site réel et les importe directement dans ce site de démonstration PrestaShop. Le chatbot travaille donc avec de vrais produits.

3. Explication de chaque fichier

Voici chaque fichier expliqué simplement, dans l'ordre logique du projet :

📦 parachatbot.php
modules/parachatbot/parachatbot.php

C'est quoi ? C'est le fichier principal du module. C'est lui qui dit à PrestaShop : "je suis un module, installe-moi".

Qu'est-ce qu'il fait ?

  • Il installe le module dans PrestaShop (fonction install())
  • Il charge le fichier CSS et le fichier JavaScript sur toutes les pages du site
  • Il injecte le widget HTML (le chat) dans le bas de chaque page (hook displayFooter)
  • Il affiche la page de configuration dans l'admin (pour saisir la clé API Gemini et voir les statistiques)
🎨 chat.tpl
views/templates/hook/chat.tpl

C'est quoi ? C'est le fichier HTML du widget chatbot. C'est ce que le visiteur voit sur l'écran.

Qu'est-ce qu'il contient ?

  • Le bouton vert "IA Conseiller Kingphar" en bas à droite
  • La fenêtre de chat qui s'ouvre quand on clique
  • Le message d'accueil du bot
  • Les 4 boutons de suggestions rapides (Peau sèche, Chute Cheveux, etc.)
  • Le champ de saisie pour taper un message
🎨 chatbot.css
views/css/chatbot.css

C'est quoi ? C'est le fichier CSS (la mise en forme). Il contient tout le design visuel du chatbot.

Qu'est-ce qu'il fait ?

  • Les couleurs (vert pour le bot, bleu pour le client)
  • La forme des bulles de messages
  • Le design des cartes produits (image + nom + prix + bouton Commander)
  • Les animations (apparition des messages, points de chargement)
  • L'adaptation pour mobile (responsive)
⚡ chatbot_v2.js
views/js/chatbot_v2.js

C'est quoi ? C'est le fichier JavaScript. C'est le moteur interactif du chatbot côté navigateur.

Qu'est-ce qu'il fait ?

  • Ouvre et ferme la fenêtre de chat quand on clique sur le bouton
  • Quand le client envoie un message → il envoie le message au serveur en AJAX (fetch) vers ajax_chat.php
  • Il attend la réponse et affiche la bulle du bot avec un effet d'écriture lettre par lettre
  • Il affiche les cartes produits recommandés (image, nom, prix, bouton "Commander")
  • Il sauvegarde l'historique dans le navigateur (localStorage) pour ne pas le perdre si on recharge la page
  • Il gère le bouton Réinitialiser (supprimer la conversation)
🔌 ajax_chat.php
modules/parachatbot/ajax_chat.php

C'est quoi ? C'est le fichier qui reçoit les messages envoyés par le JavaScript. C'est la porte d'entrée côté serveur.

Qu'est-ce qu'il fait ?

  • Il reçoit le message du client en format JSON
  • Il vérifie que le message n'est pas vide
  • Il crée les 3 services (ChatService, ProductService, AIService)
  • Il appelle ChatService->processMessage() pour traiter le message
  • Il retourne la réponse en JSON au JavaScript
Pourquoi ce fichier ? C'est un raccourci rapide. Au lieu de passer par tout le framework lourd de PrestaShop, ce fichier charge uniquement le minimum nécessaire. Résultat : le chatbot répond plus vite.
🧠 ChatService.php
classes/ChatService.php

C'est quoi ? C'est le fichier le plus important. C'est le chef d'orchestre qui coordonne tout.

Qu'est-ce qu'il fait ? Quand il reçoit un message, il fait ces étapes dans l'ordre :

  • Étape 1 — Il identifie la session du client (qui parle ?)
  • Étape 2 — Il regarde si c'est juste une salutation ("bonjour", "salam") → si oui, pas besoin de chercher des produits
  • Étape 3 — Il regarde si quelqu'un a déjà posé la même question avant → si oui, il réutilise l'ancienne réponse (économie d'API)
  • Étape 4 — Il essaie de traduire le darija en français avec un dictionnaire interne (par exemple : "lhboub" → "acné bouton visage")
  • Étape 5 — Il demande à ProductService de chercher les produits correspondants dans la base de données
  • Étape 6 — Il envoie le message + les produits trouvés à AIService (Google Gemini) pour avoir une réponse intelligente
  • Étape 7 — Il sauvegarde le message et la réponse dans la base de données
  • Étape 8 — Il retourne la réponse + les produits au JavaScript
🔍 ProductService.php
classes/ProductService.php

C'est quoi ? C'est le service qui cherche les produits dans la base de données MySQL de PrestaShop.

Qu'est-ce qu'il fait ?

  • Il reçoit des mots-clés (par exemple : "acné", "bouton", "visage")
  • Il fait une requête SQL pour chercher dans les noms et descriptions des produits
  • Il retourne une liste de produits avec : nom, prix, image, lien, stock, catégorie
Exemple simplifié de la requête SQL :
SELECT nom, prix FROM produits WHERE nom LIKE '%acné%' OR description LIKE '%bouton%'
🤖 AIService.php
classes/AIService.php

C'est quoi ? C'est le service qui communique avec Google Gemini (l'Intelligence Artificielle).

Qu'est-ce qu'il fait ?

  • Il construit un message structuré pour Gemini avec :
    • Les instructions (tu es le conseiller de Kingphar.ma, réponds en darija/français, ne recommande que les vrais produits)
    • La liste des produits trouvés par ProductService
    • L'historique de la conversation (les messages précédents)
    • Le message actuel du client
  • Il envoie tout à l'API Google Gemini via une requête HTTP (cURL)
  • Il reçoit la réponse de Gemini et la retourne
  • Si l'API est surchargée (erreur 503), il réessaie automatiquement jusqu'à 4 fois
🔀 chat.php (contrôleur)
controllers/front/chat.php

C'est quoi ? C'est le contrôleur front officiel de PrestaShop. Il fait la même chose que ajax_chat.php, mais en passant par le framework complet de PrestaShop.

En pratique, le chatbot utilise ajax_chat.php (plus rapide). Ce fichier existe comme solution de secours.

🔑 save_key.php
modules/parachatbot/save_key.php

C'est quoi ? Un petit script utilitaire pour enregistrer la clé API Google Gemini directement dans la base de données. Utile quand la page admin ne marche pas.

4. Le déroulement complet (quand un client envoie un message)

Voici exactement ce qui se passe quand un visiteur envoie un message dans le chatbot :

1
Le client tape un message (ex: "wejhi fih lhboub") et clique sur Envoyer
↓
2
chatbot_v2.js capture le message et l'envoie en AJAX (fetch POST) vers ajax_chat.php
↓
3
ajax_chat.php reçoit le message et crée les 3 services, puis appelle ChatService
↓
4
ChatService traduit le darija : "lhboub" → "acné bouton visage"
↓
5
ProductService cherche dans la base de données MySQL : WHERE nom LIKE '%acné%' → trouve les produits
↓
6
AIService envoie le message + les produits trouvés à Google Gemini → Gemini répond avec une recommandation personnalisée
↓
7
ChatService sauvegarde le message et la réponse dans la table ps_parachatbot_conversation
↓
8
chatbot_v2.js reçoit la réponse JSON et affiche la bulle du bot + les cartes produits avec images et prix
5. La base de données

Le module utilise une table MySQL pour stocker toutes les conversations :

Colonne C'est quoi ? Exemple
id_messageNuméro unique du message1, 2, 3...
session_idIdentifiant de la conversationsess_abc123_178...
roleQui a envoyé le message"user" ou "assistant"
contentLe texte du message"wejhi fih lhboub"
created_atLa date et l'heure2026-06-10 20:04:22

Pour les produits, le module ne crée pas de nouvelle table. Il utilise les tables existantes de PrestaShop (ps_product, ps_product_lang, ps_category_lang, ps_image) pour chercher les produits.
